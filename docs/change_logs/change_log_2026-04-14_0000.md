## 2026-04-14 — Eval System + Manual Testing Infrastructure

### Added
- `architect_agent/architect_eval_set.evalset.json` — Native ADK eval set (`.evalset.json` format). Wired into `adk web` Eval tab automatically. Currently 1 active case (`temporal_awareness`) — the only scenario with consistent enough tool trajectory for automated matching. Other cases removed after calibration revealed agent non-determinism.
- `architect_agent/test_config.json` — Eval criteria config for `adk eval` CLI. Threshold: `tool_trajectory_avg_score: 1.0`.
- `architect_agent/MANUAL_TEST_PLAN.md` — Step-by-step manual test plan for all 7 architect_agent scenarios. Includes exact prompts, Trace tab instructions, pass/fail criteria, and post-test golden capture workflow.
- `architect_agent/MANUAL_TEST_REPORT_TEMPLATE.md` — Fillable report template. Summary table, per-scenario detail sections, failure classification, and Phase 1 sign-off block.
- `docs/ARCHITECT_AGENT_TESTING_MANUAL.md` — Earlier version of manual testing guide (superseded by MANUAL_TEST_PLAN.md).
- `docs/change_logs/` — Changelog directory. This file is the first entry.

### Changed
- `utils/token_calculator.py` — Expanded `_PRICING` from 4 → 9 models. Added `gemini-2.0-flash-lite`, `gemini-2.5-flash-lite`, `gemini-3-pro-preview`, `gemini-3.1-pro-preview`, `gemini-3.1-flash-lite-preview`. Organised into sections (2.0 / 2.5 / 3.x preview). Preview entries flagged with `# ESTIMATE` inline comments.
- `tests/test_token_calculator.py` — Added 3 new unit tests: confirmed rate for `gemini-2.0-flash-lite`, estimate rate for `gemini-3-flash-preview`, structural guard that all `_PRICING` entries have required keys.
- `callbacks/receipt_callback.py` — Fixed timestamp injection. Replaced `append_instructions()` (accumulates across turns) with direct assignment that strips previous `[SYSTEM_TIMESTAMP:]` line before prepending fresh one. Fixes stale timestamp bug where agent read first (old) timestamp.
- `architect_agent/agent.py` — Upgraded `before_model_callback` from single callback to list: `[get_start_time_callback(), get_timestamp_inject_callback()]`. Added `get_current_datetime_tool` and `read_context_doc_tool` to tools list.
- `architect_agent/tools.py` — Added `get_current_datetime()` + `get_current_datetime_tool`. Added `read_context_doc(doc_name)` + `read_context_doc_tool`. Updated `invoke_skill` path logic: `SKILL_INDEX` stays flat, all other skills use `{skill_name}/SKILL.md` subfolder.
- `tests/test_receipt_callback.py` — Added `test_timestamp_callback_injects_timestamp` and `test_timestamp_callback_replaces_not_accumulates`.

### Fixed
- Receipt callback was failing silently for `gemini-3-flash-preview` — "Unknown model" error. Fixed by adding all known Gemini models to `_PRICING`.
- Timestamp injection was injecting a static (stale) timestamp after the first turn. Fixed by replacing instead of appending.

### Removed
- `langchain`, `langchain-openai`, `langchain-core`, `langchain-community`, `langchain-anthropic`, `langchain-text-splitters` — uninstalled from `.venv`. Not used by any current agent. Were causing `openai` version conflict after `google-adk[eval]` install.

### Infrastructure
- `google-adk[eval]` installed — adds `rouge-score`, `tabulate`, `nltk`, `scikit-learn` and other eval dependencies required for `adk eval` CLI and web UI Eval tab.
