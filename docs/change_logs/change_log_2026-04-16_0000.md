## 2026-04-16 — Custom Eval System (Stark Eval Runner)

### Added
- `architect_agent/eval_metrics.py` — 7 custom evaluators registered with ADK's `MetricEvaluatorRegistry`:
  - `StaysInRoleEvaluator` — LLM-as-Judge: agent declines off-topic requests
  - `ScopeDisciplineEvaluator` — LLM-as-Judge: agent pushes back on out-of-scope implementation
  - `SessionMemoryToolUseEvaluator` — deterministic: `read_session_memory` called
  - `SkillInvocationToolUseEvaluator` — deterministic: `invoke_skill` called
  - `ContextDocToolUseEvaluator` — deterministic: `read_context_doc` called
  - `TemporalAwarenessEvaluator` — deterministic NEGATIVE: `get_current_datetime` NOT called
  - `EngineerPromptFormatEvaluator` — deterministic: response contains TASK/SCOPE/CONSTRAINTS/DONE LOOKS LIKE
  - `_ToolPresenceEvaluator` — shared base class for the 4 tool-check metrics
- `scripts/run_stark_evals.py` — CLI eval runner. Runs custom metrics against `.evalset.json` cases. Produces timestamped `.json` + `.md` reports with box-drawing tables. Supports `--metrics`, `--baseline`, `--no-baseline` flags. Auto-discovers metrics from `DEFAULT_METRIC_EVALUATOR_REGISTRY`.
- `tests/test_stark_eval_runner.py` — 25 unit tests covering metrics discovery, file generation, baseline deltas, flags, report format, and all 4 deterministic evaluators.
- `docs/AGENT_HELPERS/RESEARCH_VERTEX_AI_EVALUATION.md` — Research report: Vertex AI Evaluation with LLM-as-a-Judge, custom `PointwiseMetric`, integration with ADK, cost estimates, code examples.
- `docs/AGENT_HELPERS/RESEARCH_VERTEX_MEMORY_BANK.md` — Research report: Vertex Memory Bank (GA), RAG Engine comparison, ADK integration via `VertexAiMemoryBankService`, MCP portability, cost analysis.
- `README.md` — Replaced placeholder with full landing page: ASCII architecture diagram, quick start, agent table, repo tree, capabilities.
- `docs/HARNESS_v1_DOCS/01_ARCHITECTURE.md` through `09_OPERATIONS.md` — 9 documentation files covering system design, file structure, agents, tools, skills, session memory, prompts, testing, and operations.

### Changed
- `architect_agent/__init__.py` — Added 3 lines: import + call to `register_custom_metrics()` at import time. Custom metrics register before `adk web` starts serving.

### Discovered (ADK v1.13.0 Limitation)
- ADK web UI Eval tab frontend is hardcoded to 2 metrics (`tool_trajectory_avg_score`, `response_match_score`) in compiled Angular JavaScript. The `MetricEvaluatorRegistry` backend and `/metrics-info` endpoint work correctly and return all registered metrics. The frontend ignores the backend and uses a static array. Custom metrics work via the Python API (`LocalEvalService`, `run_stark_evals.py`) but cannot appear in the UI dropdown. This is not fixable without modifying ADK's compiled frontend.

### Research Findings
- **Vertex AI Evaluation:** `tool_trajectory_avg_score: 1.0` is the wrong metric for reasoning agents. Vertex AI Evaluation with LLM-as-a-Judge (`PointwiseMetric`, adaptive rubrics, custom criteria) is the correct approach. Cost: ~$0.05 per 100 evals.
- **Vertex Memory Bank:** GA since late 2025. First-class ADK integration via `VertexAiMemoryBankService` + `PreloadMemoryTool`/`LoadMemoryTool`. Complementary to RAG Engine — Memory Bank for user personalization, RAG Engine for knowledge grounding. Community MCP server exists for portability.
